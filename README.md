Sample Gradle project (with both Groovy and Kotlin flavors) showing [NullAway](https://github.com/uber/NullAway) configured with the JSpecify mode.
A nullness issue is visible in IDEs with support for JSpecify or when running `./gradlew build`.

For Maven, see a sample `pom.xml` [here](https://github.com/sdeleuze/jspecify-nullway-demo/blob/maven/pom.xml) in the [maven branch](https://github.com/sdeleuze/jspecify-nullaway-demo/tree/maven).
